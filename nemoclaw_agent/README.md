# nemoclaw_agent

Containerized NemoClaw setup that runs the interactive onboarding wizard inside Docker,
persisting all generated config into `./nemoclaw-data/` (tracked in this repo).

## First-time onboarding

```bash
# Build the image
docker compose build

# Run the interactive onboarding wizard
docker compose run --rm nemoclaw nemoclaw onboard
```

The wizard runs inside the container attached to your terminal. All config it generates
is written to `/root/.nemoclaw` inside the container, which is bind-mounted to
`./nemoclaw-data/` on your host.

## Subsequent runs

Once onboarded, the config is already in `./nemoclaw-data/`. You can start the agent
directly:

```bash
docker compose run --rm nemoclaw nemoclaw start
```

Or drop into a shell to inspect/run commands manually:

```bash
docker compose run --rm nemoclaw bash
```

## Directory structure

```
nemoclaw_agent/
├── Dockerfile          # Installs nemoclaw CLI
├── docker-compose.yml  # Wires up volumes and TTY
├── README.md           # This file
└── nemoclaw-data/      # Generated by onboard wizard (gitignored)
    └── ...
```

> **Note:** Add `nemoclaw-data/` to `.gitignore` if it contains API keys or secrets.
